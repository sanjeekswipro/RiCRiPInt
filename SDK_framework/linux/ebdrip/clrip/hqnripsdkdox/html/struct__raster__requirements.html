Detailed Description

Read/write structure used during the DeviceIOCtl_RasterRequirements and DeviceIOCtl_RenderingStart IOCTL calls.

Field Documentation

uint32 band_height

Height of each band in lines.

size_t band_size

The size of the entire band buffer.

uint32 band_width

Width of each band.

uint32 components

Number of color components overall.

uint32 components_per_band

Number of color components present in each band.

int32 eraseno

Page generation ID.

int32 handled

Flag which is set to FALSE by the caller, and set to TRUE by the callee if it has completed. If not, the caller will keep calling with the same data until either this flag is set to TRUE or the callee returns an error.

int32 have_framebuffer

Set to TRUE by RasterRequirements if it has (or is able to promise by the time DeviceIOCtl_GetBufferForRaster is called) a complete framebuffer.

int32 interleavingStyle

Interleaving style.

uint32 line_bytes

Number of bytes that the RIP will write for each line of a band.

uint32 minimum_bands

The minimum number of raster destination bands that the RIP requires.

uint32 nChannels

Number of color channels.

uint32 page_height

Page height in pixels.

uint32 page_width

Page width in pixels.

void* scratch_band

Scratch buffer handed by the core to RenderingStart.

size_t scratch_size

Set by RasterRequirements to the size of a scratch buffer requested from the core.
