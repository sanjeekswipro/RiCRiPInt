Detailed Description

This file provides the information required to set up devices outside the rip.

HopeName
COREinterface_control!swdevice.h(EBDSDK_P.1)

Copyright (C) 1992-2013 Global Graphics Software Ltd. All rights reserved. Global Graphics Software Ltd. Confidential Information.

The typical method will be to:

  • define a series of routines to manage devices of each required type;
  • setup a static structure forming the device type. For example:
 *    static DEVICETYPE my_device_type = {
 *       my_device_type_number,
 *       ...
 *       & my_device_read_file,
 *        ...
 *     };
 *
  • add the address of the structure to a static, null terminated array of such structures. Remember that an os device type of 0 must always be provided. e.g.:
 *    static DEVICETYPE * all_my_devices [] = {
 *      & my_os_device_type,
 *      ...
 *      & my_device_type,
 *      ...
 *      NULL
 *    };
 *
  • put the address of this array into the SWSTART array with the correct tag (see swstart.h) and pass the array to the rip via SwStart()

OEM's please note: except for OS_DEVICE_TYPE and any numbers specifically mentioned in product documentation (which will normally be listed above), you must ONLY USE DEVICE TYPE NUMBERS IN THE RANGE ALLOCATED TO YOU BY HARLEQUIN. Current or future releases of the Harlequin RIP may fail if you do not.

Core RIP Device Interface Version information

This file is exported to PS DeviceType plugins, and they expect binary compatibility with this interface as far as possible.

If this interface changes in ANY non-backwards compatible way, the Major version number will be incremented.

For any other changes, which can still support plugins with the same major version number, only the minor version number will be incremented.
