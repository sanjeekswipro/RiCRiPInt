Detailed Description

This header file provides definitions for the structured data interface to the core RIP.

The structured data interface is a callback-based interface used to access parameters passed to a module.

A sw_datum can represent a boolean, integer, float, string, name, array, stack or dictionary. It can represent a Postscript object and act as a proxy for it to allow manipulation of Postscript state, or it can be an independent module-created 'raw' value.

If 'raw' values are insufficient, implementing the sw_data_api interface allows the content of complex types such as arrays and dictionaries to be created on demand rather than in advance. The RIP uses this approach to grant access to Postscript objects through particular interfaces.
